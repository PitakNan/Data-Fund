## 3. วิธีต่อ Power BI

1. เปิด **Power BI Desktop** → Home → **Get Data → Text/CSV** → เลือก `combined_fact.csv`
   - ตรวจ File Origin ให้เป็น **65001: Unicode (UTF-8)** (ภาษาไทยจะไม่เพี้ยน เพราะไฟล์เป็น UTF-8-BOM อยู่แล้ว)
2. ตั้งชนิดข้อมูล: `date_admit`,`date_dsc` = Date ; `adjrw`,`amount_baht`,`age`,`qty_item` = Decimal/Whole Number ; ที่เหลือ = Text
3. (แนะนำ) สร้าง **Calendar table** จาก `date_admit` เพื่อทำเทรนด์รายเดือน/ปีงบ
4. สร้าง Measure หลัก (DAX):

```DAX
จำนวนเคส     = COUNTROWS('combined_fact')
จำนวนคน      = DISTINCTCOUNT('combined_fact'[pid_hash])
ยอดเงินรวม   = SUM('combined_fact'[amount_baht])
adjRW เฉลี่ย = AVERAGE('combined_fact'[adjrw])
ค่าเฉลี่ยเงินต่อเคส = DIVIDE([ยอดเงินรวม],[จำนวนเคส])
```

## 4. โครงหน้า Dashboard ที่แนะนำ

**Slicer ร่วมทุกหน้า:** `ปีงบฯ`, `prov_name`, `ระดับหน่วย`, `ประเด็น`

- **หน้า 1 — ภาพรวมเขต:** การ์ด KPI (เคส/คน/เงินรวม) · บาร์ 8 จังหวัด · เทรนด์ปีงบ 66→69 · โดนัทสัดส่วน 5 ประเด็น
- **หน้า 2 — Heat map รพ. × ประเด็น:** Matrix รพ.(แถว) × ประเด็น(คอลัมน์) ใส่สี เพื่อหา รพ.ที่ติดหลายธง
- **หน้า 3 — adjRW<0.5:** Top รพ./pdx ความซับซ้อนต่ำ · แยก `LOS_Group`
- **หน้า 4 — ส่งต่อออกนอก สป.สธ.:** Sankey/Matrix `ref_from_name` → `hcode_name` · เงินที่รั่วออกนอกระบบ
- **หน้า 5 — NB <1,500g:** จำนวนทารกตาม รพ./ระดับหน่วย (ดูศักยภาพ NICU) · Top pdx
- **หน้า 6 — จิตเวช homeward:** เคส/เงินตาม รพช. · Top pdx
- **หน้า 7 — TKA:** เคสตามจังหวัด/รพ. · สัดส่วน revision · กลุ่มอายุ–เพศ

## 5. ข้อควรรู้เรื่องคุณภาพข้อมูล
- ตัด header เกินของ TKA ออกแล้ว 1 แถว
- ปีงบฯ ของ TKA คำนวณจากวันบริการ (ปีงบเริ่ม ต.ค.) เพราะไฟล์ต้นทางไม่มีคอลัมน์ปีงบ
- ไฟล์ homeward จิตเวช แปลงจาก long → wide แล้ว (รวม adjrw และเงินไว้แถวเดียวต่อเคส)
- มี ~13 แถวจังหวัดปลายทาง "กรุงเทพฯ" ในประเด็นส่งต่อออกนอก สป.สธ. (ถูกต้อง = ส่งไป รพ.ใน กทม.)
